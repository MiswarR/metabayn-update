Metabayn Studio – Prompt Arsitektur & Alur Fitur Lengkap
========================================================

Tujuan dokumen:
- Menjelaskan secara ringkas namun lengkap bagaimana seluruh komponen Metabayn Studio bekerja.
- Menjadi “prompt sistem” bagi AI lain agar mengerti konteks aplikasi, fitur, dan alur proses.
- Tetap akurat dengan kode saat ini (desktop Tauri + React + Rust, backend Cloudflare, integrasi AI).

--------------------------------------------------------
1. Gambaran Umum Aplikasi
--------------------------------------------------------

Metabayn Studio adalah aplikasi desktop (Tauri + React + Rust) untuk fotografer / videografer stok yang mengotomatisasi:
- Pembuatan metadata lengkap (judul, deskripsi, kata kunci, kategori Shutterstock).
- Seleksi kualitas (accept / reject) berdasarkan kriteria teknis dan konten.
- Penulisan metadata langsung ke file (Exif/XMP) dan pembuatan CSV untuk upload stok.
- Deteksi duplikat visual.
- Klastering foto/video dengan AI (AI clustering).
- Pengelolaan saldo token & mode langganan API key pribadi.

Arsitektur utama:
- Frontend: React (Vite) yang berjalan di dalam WebView2 via Tauri.
- Desktop shell: Tauri (Rust) untuk akses file system, ExifTool, FFmpeg, dan updater.
- Backend server: Cloudflare Workers (metabayn-backend) dengan DB untuk user, model, harga, voucher, dsb.
- AI providers:
  - OpenAI (gpt-4o, gpt-4.1, gpt-4o-mini, dsb).
  - Google Gemini (gemini-3.0, 2.5, 2.0, 1.5, dll).
  - Provider lain sesuai konfigurasi model.

Mode koneksi AI:
1) Mode "server":
   - Aplikasi mengirim permintaan ke backend Metabayn.
   - Backend yang berkomunikasi dengan provider AI.
   - Pembayaran berdasarkan saldo token internal.

2) Mode "direct" (API key pribadi):
   - Aplikasi mengirim ke provider AI langsung menggunakan API key pengguna.
   - Tidak mengurangi saldo token di backend.
   - Cocok untuk pengguna yang ingin pakai kuota pribadi di OpenAI/Gemini.

3.3. Dashboard – Proses Utama Pengguna
--------------------------------------

File: src/pages/Dashboard.tsx

Fungsi global:
- Menampilkan:
  - Tombol Mulai/Jeda/Stop.
  - Progress bar (total, success, reject, failed).
  - Panel log real-time.
  - Tombol Settings, Deteksi Duplikat, AI Cluster, CSV, Top Up, Admin.
- Mengatur bahasa (EN/ID) dan teks UI (melalui translations.ts).
- Memantau saldo token:
  - Saat token ada → panggil checkServerAdmin + refreshBalance.
  - Poll saldo tiap 30 detik.
- Menangani update otomatis aplikasi:
  - checkUpdate → jika ada versi baru → installUpdate → relaunch.

Alur proses utama "Mulai":
1) Validasi minimal:
   - Input/Output folder terisi.
   - User login dan (jika mode server) saldo cukup.
2) Membaca settings:
   - server_url, default_model, max_threads, retry_count, filter dan opsi lain.
3) Memindai folder input:
   - invoke('scan_folder') → Rust → mengembalikan daftar file yang valid (foto/video).
4) Inisialisasi statistik dan log.
5) Menentukan concurrency (jumlah file paralel):
   - Berdasarkan max_threads dan jumlah core CPU user.
6) Untuk setiap file:
   - Cek apakah sudah pernah diproses (cek file di output folder / approved / rejected).
   - Jika sudah ada → tulis log "[Skipped]" dan lanjut ke file berikutnya.
   - Jika belum:
     - Tulis log "Processing {filename}...".
     - Panggil generate_metadata_batch via invoke:
       - Request berisi:
         - files: [path file].
         - model: model efektif (dipilih tergantung provider).
         - token: kosong jika mode direct, atau token auth jika mode server.
         - retry_count, batas kata/jumlah karakter title/description, keywords, banned_words.
         - max_threads (untuk sisi Rust).
         - connection_mode: "server" atau "direct".
         - api_key: hanya diisi jika mode direct.
         - provider: "Gemini", "OpenAI", dll.
7) Hasil generate_metadata_batch:
   - Untuk setiap Generated:
     - Jika g.source == 'error' atau title == "ERROR" atau selection_status == 'rejected':
       - Log sebagai gagal/ditolak, simpan detail (termasuk token usage & cost jika ada).
       - File bisa dipindah ke folder rejected (tergantung pengaturan).
     - Jika sukses:
       - Log sebagai sukses.
       - Tulis metadata ke file melalui Rust (ExifTool).
       - Pindahkan file ke folder output/approved jika pengaturan selection aktif.
8) Update statistik dan progress bar:
   - total, done, success, failed, rejected.

Fitur lain di Dashboard:
- Generate CSV:
  - Pilih folder input.
  - invoke('generate_csv_from_folder') → Rust:
    - Membaca metadata file.
    - Jika beberapa field Shutterstock kosong → memanggil AI untuk mengisi.
    - Menulis CSV di folder tersebut.
- Duplicate scan:
  - Pilih folder.
  - invoke('detect_duplicate_images') → Rust:
    - Analisis hash gambar.
    - Menentukan duplikat berdasarkan threshold.
    - Opsional auto delete.
- AI clustering:
  - invoke('run_ai_clustering'):
    - Rust menjalankan skrip Python ai_cluster.py sebagai proses eksternal.
    - Log progres ke event "ai_cluster_log".

4.2. Fungsi generate_metadata_batch
-----------------------------------

File: src-tauri/src/api.rs

- Tauri command: generate_metadata_batch(req: BatchReq) → Result<Vec<Generated>, String>
  - BatchReq berisi:
    - files, model, token, retries.
    - batas panjang title/description/keywords.
    - banned_words, max_threads.
    - connection_mode, api_key (opsional), provider.
- Fungsi ini memanggil crate::metadata::generate_batch(&req).

File: src-tauri/src/metadata.rs

Fungsi utama:
- generate_batch(req: &BatchReq) → Result<Vec<Generated>>
  - Mengambil settings untuk selection & filter.
  - Menyortir daftar file (natural sort).
  - Untuk setiap file:
    - Memastikan file ada.
    - Memanggil prepare_image_data:
      - Jika video → ekstrak frame dengan FFmpeg (resources/ffmpeg).
      - Jika gambar → baca, retry bila gagal, resize jika perlu, re-encode JPEG, Base64.
    - Menyusun prompt AI:
      - Prompt selection (jika selection_enabled).
      - Prompt utama untuk metadata (title, description, keywords, kategori).
      - Prompt tambahan untuk flags kualitas / konten.
    - Memanggil call_ai_base:
      - Menghitung token usage dan cost jika provider mendukung.
    - Mem-parse JSON hasil AI menjadi struct Generated:
      - title, description, keywords (vector string).
      - category.
      - selection_status ("accepted"/"rejected").
      - penanda kualitas / jenis kegagalan (untuk tagging).
      - usage tokens dan estimasi cost.
    - Menjamin judul unik (ensure_unique_title).
  - Mengembalikan vector Generated ke Dashboard.

Fungsi call_ai_base:
- Menentukan URL endpoint berdasarkan provider/model/mode:
  - Mode server:
    - Memanggil backend API (metabayn-backend) atau server lain.
  - Mode direct:
    - Jika provider Gemini / API key Google:
      - Panggil endpoint Gemini yang kompatibel.
    - Jika provider OpenAI:
      - Panggil https://api.openai.com/v1/chat/completions.
- Menangani error jaringan, status non-2xx, dan retry (untuk 429/404 tertentu).
- Mengembalikan:
  - konten teks (biasanya JSON string).
  - usage (TokenUsage) opsional.
  - model yang dipakai (actual).
  - cost estimasi opsional.


4.3. Generate CSV & Penulisan Metadata
--------------------------------------

Masih di metadata.rs:

- Fungsi generate_csv_from_folder:
  - Membaca file CSV atau metadata file di folder input.
  - Untuk entry yang belum punya kategori/editorial/mature/illustration:
    - Mengirim ke AI dengan prompt khusus “Shutterstock metadata filler”.
    - Mengisi field yang kosong dengan hasil AI.
  - Menulis atau mengupdate CSV hasil.

- Di dalam loop pengisian metadata, setelah AI memberikan JSON:
  - Aplikasi membuat struktur JSON baru (categories, editorial, mature, illustration).
  - Menulis ke file menggunakan ExifTool:
    - Memanggil exiftool sebagai sidecar bawaan Tauri.
    - Menggunakan loop retry dengan batas dan jeda 500ms.

5. Backend Metabayn (Cloudflare Workers)
--------------------------------------------------------

5.1. Handler AI (ai.ts)
-----------------------

Lokasi: metabayn-backend/src/handlers/ai.ts

Peran:
- Menerima permintaan AI dari aplikasi (mode server).
- Mengelola antrian dan concurrency via aiQueue:
  - enqueue, setConcurrencyLimit.
  - Membatasi jumlah job paralel untuk menghindari rate limit provider.
- Menentukan model dan provider yang digunakan:
  - MODEL_CONFIG.models dari database (model_prices).
  - fallback chain (misalnya: model utama, lalu model backup lain).
- Untuk setiap job:
  - Mencatat waktu mulai dan batas durasi (MAX_DURATION).
  - Mencoba memanggil model dalam fallback chain:
    - Menentukan provider dari konfigurasi atau menebak dari nama model (gemini, claude, openai).
    - Menyusun payload yang sesuai dengan provider (OpenAI / Gemini / Anthropic).
    - Menangani error:
      - Network error, timeout.
      - Error 429 (rate limit), 5xx (server error).
    - Menerapkan retry logis jika provider mengizinkan.
  - Mengembalikan:
    - hasil teks JSON metadata,
    - usage tokens,
    - estimasi cost (berdasarkan harga input/output per 1M token dan profit multiplier).


5.2. Handler Admin & Model Pricing (admin.ts)
---------------------------------------------

Peran:
- CRUD model dan harga di tabel model_prices:
  - provider (openai, gemini, dsb.)
  - model_name (gpt-4o, gemini-2.5-flash, dll.)
  - input_price, output_price (per 1M token).
  - profit_multiplier.
  - active (on/off).
  - fallback_priority.
- Menyediakan endpoint untuk:
  - Mengambil daftar model aktif untuk ditampilkan di Settings.
  - Menyimpan perubahan harga, mengupdate model, dsb.
- Mengisi data awal dengan daftar model OpenAI + Gemini terbaru.


5.3. Handler Auth, Voucher, Subscription
----------------------------------------

auth.ts:
- Login:
  - Verifikasi email/password.
  - Cek status user (pending/aktif).
  - Khusus email admin tertentu, set is_admin=1.
  - Cek subscription_active dan subscription_expiry:
    - Jika sudah lewat → matikan status subscription.
- Menghasilkan token sesi (JWT / token custom) yang digunakan app.

voucher.ts:
- Memetakan produk dari payment gateway ke:
  - Voucher token (menambah saldo token).
  - Subscription API key mode (mengaktifkan mode direct untuk jangka waktu tertentu).
- Menangani redeem voucher, validasi, dan update DB.

8. Prinsip Desain & Batasan Penting
--------------------------------------------------------

- Jangan mengubah hasil metadata yang sudah terbukti baik:
  - Prompt utama untuk title/description/keywords harus tetap konsisten.
  - JSON schema yang diharapkan AI untuk dihasilkan tidak boleh diubah sembarangan.

- Seleksi (accept/reject) dan filter kualitas:
  - Bergantung pada flag yang dikembalikan AI (gagal karena blur, watermark, brand, gibberish, dsb).
  - Perubahan pada parsing atau mapping bisa mempengaruhi jumlah file yang ditolak/diterima.

- Kinerja:
  - Concurrency dikontrol dari Settings (max_threads) + hardwareConcurrency.
  - Rust/Tauri menggunakan tokio dan Semaphore untuk menghindari over-subscription.
  - Beberapa operasi I/O (file, ExifTool, FFmpeg) sudah memiliki retry + jeda (sleep) untuk stabilitas.

- Keamanan:
  - Token auth dan API key disimpan terenkripsi di sisi desktop.
  - Rate limiting di SecurityService dan di backend mencegah abuse.
  - AuditService mencatat event penting untuk debugging / compliance.

9. Cara Menggunakan Prompt Ini
--------------------------------------------------------

Prompt ini dapat diberikan ke model AI lain sebagai konteks sistem, misalnya:

- "Kamu adalah asisten teknis untuk aplikasi Metabayn Studio. Gunakan spesifikasi berikut untuk menjawab pertanyaan, merancang fitur, atau menganalisis bug. Jangan menyimpang dari alur yang dijelaskan, dan pastikan semua perubahan tidak merusak fungsi-fungsi yang sudah berjalan dengan baik."

Dengan prompt ini, AI diharapkan:
- Memahami alur lengkap dari user → desktop → backend → AI provider.
- Mengerti peran masing-masing komponen (React, Tauri, Rust, Cloudflare Workers).
- Dapat memberi saran atau perubahan kode yang tetap sejalan dengan desain aplikasi.
